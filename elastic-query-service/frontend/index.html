Panel de documentos — elastic-query-service

Cubre los 3 endpoints de ElasticDocumentController que hoy no tienen ninguna interfaz: listar todos los documentos y obtener uno por id (v1/v2). La búsqueda por texto ya tiene su propia UI en elastic-query-web-client / elastic-query-web-client-2 / reactive-elastic-query-web-client, así que no se duplica aquí.

Configuración

El token no se guarda (ni localStorage ni sessionStorage): solo vive en memoria mientras esta pestaña siga abierta, porque es un JWT real de Keycloak. Pégalo de nuevo si recargas la página. Estos 3 endpoints exigen anyRequest().authenticated() vía OAuth2 resource server (issuer .../realms/microservices-realm); además getDocumentById exige permiso concreto sobre ese id o el rol APP_SUPER_USER_ROLE (ver QueryServicePermissionEvaluator) — sin ese rol/permiso, esperá un 403.

1. Listar todos los documentos

GET /documents

2. Obtener documento por id

GET /documents/{id} — v1/v2 vía negociación de contenido (header Accept)
